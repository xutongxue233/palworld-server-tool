# Changelog

本文件记录 palworld-server-tool 的重要功能、修复与安全变更。

## [Unreleased] - 2026-07-10

### 新增

- 完整接入 Palworld 官方服务端 REST API，包括服务器设置、世界 Actor 数据、立即保存世界和强制停止服务器。
- 新增 `/api/server/settings`、`/api/server/game-data`、`/api/server/save` 和 `/api/server/stop` 管理接口。
- 桌面端控制中心新增服务器设置查看、保存世界、优雅关服和强制停止操作。
- 白名单新增跨平台 `user_id` 支持，不再依赖 Steam ID 作为唯一身份标识。
- 新增 REST API、JWT、归档解压、服务器接口和白名单服务测试。
- 新增 GitHub Actions，自动执行后端测试、静态检查、漏洞扫描、前端依赖审计、Lint 和构建。
- 新增开发环境资源占位嵌入方案，使干净仓库无需预生成前端与地图资源即可测试和编译。

### 变更

- 重构 Palworld REST 客户端，使用逐请求上下文超时、统一 Basic Auth、类型化响应和结构化错误。
- 补全服务器描述、世界 GUID 和据点数量等官方 API 字段。
- 玩家踢出、封禁和解封统一优先使用官方 `userId`，并保留 Steam 平台兼容路径。
- JWT 实现迁移至 `github.com/golang-jwt/jwt/v5`，并严格限制为 HMAC SHA-256。
- Docker 客户端迁移至新版 `github.com/moby/moby/client` 和独立 API 模块。
- Go 最低版本升级至 1.25；容器前端构建升级至 Node.js 22 和 pnpm 9.15.9。
- 前端升级至 Vite 6 和 Vue I18n 11，并将构建工具移动到开发依赖。
- Docker 和 Makefile 的生产构建统一使用 `assets` 构建标签。
- 更新 Swagger 文档、README 功能说明和多语言界面文本。

### 修复

- 修复配置加载前初始化 JWT 密钥导致密钥为空、令牌可被伪造的严重认证问题。
- 修复 REST 客户端修改共享 HTTP Client 超时时间造成的数据竞争。
- 修复仅接受 HTTP 200、忽略其他成功状态码和请求构造错误的问题。
- 修复玩家同步失败时误判全部玩家下线，并可能触发错误广播或白名单踢出的行为。
- 修复调度器全局实例未保存，导致停止任务时实际调度器仍继续运行的问题。
- 修复优雅关服参数校验、可选消息以及玩家操作消息透传。
- 修复归档解压中的 Zip Slip、Tar Slip、文件截断、权限和资源释放问题。
- 修复下载请求无超时、未检查 HTTP 状态码以及失败文件残留问题。
- 修复前端伪造远程 IP 请求头、空令牌认证头、网络错误处理和查询参数编码问题。
- 修复前端令牌过期解析、翻译函数未定义、登录弹窗引用、地图循环键和 Vue 深层选择器语法问题。
- 清理前端未使用导入、变量和旧辅助函数，使 ESLint 达到零错误、零警告。

### 安全

- 主 Go 项目的 `govulncheck` 可达漏洞降为 0。
- 主 Web 项目的完整 npm 依赖审计降为 0。
- 登录密码比较改为恒定时间比较，JWT 校验强制检查签名算法。
- CI 持续执行 Go 可达漏洞扫描和前端完整依赖审计。

### 已知限制

- 外部 `pal-conf` 子模块仍包含上游依赖安全告警，主要来自自动安装的 Next.js peer 和旧构建工具链；需要在该子模块上游升级后更新引用。
- 主界面生产包仍约为 3.2 MB，后续应对地图、玩家详情、物品数据和管理模块进行按需加载与代码拆分。
