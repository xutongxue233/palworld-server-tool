# Palworld Server Tool v1.9.1

此版本继续面向 Palworld 1.0.0，改进 Web 管理模式的首次设置与密码维护流程。新安装不再依赖启动日志中的随机密码，管理员可直接在浏览器中完成初始化，并在登录后随时修改密码。

## 主要更新

- 新安装不再自动生成或在日志中打印 Web 管理密码。
- 首次打开 Web 界面时会显示不可跳过的密码设置窗口，要求输入并确认 8-512 个字符的管理密码；成功后直接进入管理模式。
- 已登录用户可从顶部管理菜单修改密码。新密码立即写入 `pst.db` 并生效，无需重启 PST。
- 修改密码会轮换当前 JWT，并使其他已登录的管理会话失效。
- 新增只读认证状态接口和一次性密码初始化接口；未设置密码时拒绝登录和 JWT 签发。
- Fleet 节点令牌不能初始化或修改 Web 管理密码；通过 `WEB__PASSWORD` 配置的部署继续由环境变量管理，界面会禁用修改入口。
- 地图资源构建兼容 PalDB 当前的世界坐标格式，确保发布包能够刷新最新高塔与传送点数据。
- 同步更新三语界面、README、更新记录和 Swagger API 文档。

## 升级说明

- 从 v1.9.0 升级时，`pst.db` 中已有的 `web.password` 会继续使用，不会被清除或重新生成。
- 升级后可登录管理模式，从顶部“管理模式”菜单选择“修改管理密码”。修改立即生效，其他浏览器中的旧登录状态需要使用新密码重新登录。
- 新安装只需启动 PST 并打开 Web 页面，按提示设置密码；无需再查看控制台日志获取初始密码。
- 若部署显式设置了 `WEB__PASSWORD`，请继续通过环境变量修改密码并重启 PST。
